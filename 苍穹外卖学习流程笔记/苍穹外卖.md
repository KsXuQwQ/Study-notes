## 整体项目结构

| sky-take-out | maven父工程，统一管理依赖版本，聚合其他子模块                  |
| ------------ | ------------------------------------------ |
| sky-common   | 子模块，存放公共类，例如：工具类，常量类，异常类等                  |
| sky-pojo     | 子模块，存放实体类、==VO、DTO==等                      |
| sky-server   | 子模块，后端服务，存放配置文件、Controller、Service、Mapper等 |
|              |                                            |
==VO：视图对象，为前端展示数据提供的对象==，前端的表单和数据需要后端提供，后端提供的数据封装给到前端的这个对象即为VO

==DTO：数据传输对象，通常用于程序中各层之间转递数据==，比如前端传入数据，那么后端需要接收数据并且封装Java对象，这个时候可以使用DTO

POJO：（从语法上来看，VO，DTO都是POJO，项目细化做了区分）普通Java对象，只有属性和对应的getter和setter
Entity：实体，通常与数据库的表对应

## 前后端联调

Controller层 --> Service层 --> Mapper层

Controller层
1. 接受并封装参数
2. 调用service层方法查询数据库
3. 封装结果并响应

Service层
1. 调用mapper查询数据库
2. 密码比对
3. 返回结果

Mapper层
发送数据库操作语句，操作数据。

### 关于Nginx反向代理

Nginx：将前端发送的动态请求由 Nginx转发到后端

Nginx反向代理的好处：

- 提高访问速度  （客户端再次请求相同内容时，Nginx可以直接从自己的缓存中返回数据，从而避免重复向后端服务器发起请求，达到加速访问和减轻后端压力的效果）
- 负载均衡： 将大量的请求按照指定方式均衡分配给集群中的每台服务器
- 保证后端服务器安全：事实上前端不能直接访问后端（不安全），而是需要先发给nginx走内网访问后端服务
## Nginx反向代理的配置方式

```
server{
       listen 80;
       server_name localhost;
       
       location /api/{ 
                      proxy_pass http://localhost:8080/admin/ #反向代理 （admin后有 / 意味着匹配Api替换后，在其后加上替换前的请求）
       }

}
```
- **`listen 80;`**：Nginx 监听本机的 80 端口（HTTP 默认端口）。
- **`server_name localhost;`**：只有当你通过 `localhost` 这个域名（或 127.0.0.1）访问时，这个配置才会生效。多个相同端口号的情况下，指定进入哪个配置时，通过此命令区分。
- location /api/：当请求匹配到 `/api/` 路径时，触发转发
## Nginx负载均衡配置

```
upstream webservers（这个就是服务器需要均衡访问的地址）{
    server 192.168.100.128:8080;
    server 192.168.100.129:8080;
}

server{
    listen 80;
    server_name localhost;

    location /api/ {
    proxy_pass http://webservers（与upsatream对应，不限制名字只要对应）/admin; #负载均衡
    }
}
```
