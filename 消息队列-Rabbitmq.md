# 什么是消息队列

消息队列：使用于在应用程序之间传递消息的通讯方式，消息队列允许应用程序异步地发送和接受消息，并且不需要直接连接到对方

队列：可以说是一个数据结构，可以存储数据，先进先出

# 消息队列的应用场景

传统模式：
订单系统通知库存系统削减库存，顺便通知物流系统
引入MQ后:
订单系统去MQ，接着MQ通知库存系统削减，顺便通知物流系统

MQ作为中间件，实现了解耦，且在库存发生异常的时候（传统模式，会直接告知库存系统导致错误），库存会先告知MQ，MQ再告知订单系统，待异常消失再会削减库存，并提醒物流，提高了系统的容错

且MQ可以将大量请求分成少量请求多次访问，保持稳定，
MQ还可以将请求同时发送给两个请求，以减少整体访问时间

缺点：
降低了系统的可用性，系统引用的外部依赖多，越容易挂掉
