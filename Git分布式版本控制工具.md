# 概述
## 开发中的实际场景

> 场景一：备份
> 	 如果在某一天你的电脑损坏或者必要文件丢失了，那么可以通过git来备份通过上传仓库，github 再从网络拉取回来完成备份。
> 场景二：代码还原
> 	假如公司要求你开发复杂代码，许多天后你终于开发完成，但是代码已经面目全非了，git可以回溯你每次提交的版本历史，从而进行版本还原。
> 场景三：协同开发
> 	项目很大的情况需要很多人维护，所以为了每个人都能精准的维护自己负责的代码，git可以进行协同开发，避免开发同一个功能，从而实现每个人从一个项目分别实现不同功能。

## 版本控制器的方式
a、集中式版本控制工具
	版本库是集中存放在中央服务器的，团队每个人工作时，都需要从中央服务器下载代码，必须联网才能完成工作。个人修改后提交到中心版本库。
	举例 ：SVN和CVS（过时了）
b、分布式版本控制工具
	分布式版本系统没有 中央控制器，每个人的电脑上都是一个完整的版本库，这样工作的时候无需联网，因为版本库就在本地，多人协作的时候只需要将修改部分推送给对方，就能互相看到对方的修改。
	举例：git
